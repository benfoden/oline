# Contributing to Oline

Thank you for your interest in contributing to Oline!

## Philosophy

Oline is built on a simple conviction: people deserve tools that help them live according to their own values, not tools that exploit their psychology. We welcome contributions that advance this mission — whether that's improvements to the conversation engine, better question frameworks, documentation, or tooling.

## Contributor License Agreement

This project uses the **Developer Certificate of Origin (DCO)** instead of a traditional CLA. It's simpler and lighter-weight.

### Sign-off Requirement

Every commit must include a `Signed-off-by` line, which certifies that you have the right to submit the contribution and that you agree to the DCO:

```
Signed-off-by: Your Name <your@email.com>
```

You can add this automatically to your commits by using:

```bash
git commit --signoff -m "Your commit message"
```

By making a contribution to this project, you certify that:

1. The contribution was created in whole or in part by you and you have the right to submit it under the open source license indicated in the project; or
2. The contribution is based upon previous work that, to the best of your knowledge, is covered under an appropriate open source license and you have the right under that license to submit that work with modifications, whether created in whole or in part by you, under the same open source license (unless you are willing to change to a different license); or
3. The contribution was provided directly to you by some other person who certified (a) or (b) above, and you have not modified it.
4. You understand and agree that this project and the contribution are public and that a record of the contribution (including all personal information you submit with it, including your sign-off) is maintained indefinitely and may be redistributed consistent with this project or the open source license(s) involved.

### Dual Licensing Note

By contributing to Oline, you agree that contributions may be used in both the open source community version and any commercial or premium versions of Oline. This ensures the project can grow and sustain itself while remaining open. If this is unacceptable to you, please do not contribute.

## How to Contribute

1. **Fork** the repository
2. **Create a branch** for your feature or fix (`git checkout -b feature/your-feature-name`)
3. **Make your changes** with clear, thoughtful commit messages
4. **Add tests** if applicable — Oline is a conversation engine, so quality matters
5. **Sign off** your commits with `git commit --signoff`
6. **Push** to your fork and **submit a pull request**

## What We're Looking For

- Improvements to the Socratic question framework
- Better handling of deflections and contradictions
- Documentation improvements
- Bug fixes
- Multilingual support contributions
- Tools and scripts for the community

## What We Generally Won't Accept

- Features that contradict the core philosophy (manipulation, gamification, dark patterns)
- Contributions that significantly increase complexity without proportional benefit
- Code that compromises user privacy

## Getting Help

- Open an issue for bugs or feature requests
- Start a discussion for general questions

## Code of Conduct

Be kind, respectful, and constructive. We're all here to help people live better lives — let's model that in how we work together too.
